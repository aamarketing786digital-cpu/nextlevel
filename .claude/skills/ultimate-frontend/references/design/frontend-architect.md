# Frontend Architect Skill

## Philosophy
1. **Epicenter of Design**: Define ONE core interaction that defines the page.
2. **Motion Choreography**: Plan animations before coding (Scroll vs Micro-interaction).
3. **Design Tokens**: Use semantic variables (`--surface-glass`) over hex codes.
4. **Bold Choices**: Avoid generic layouts; use asymmetry, depth, and texture.

## Process
1. **Phase 0**: Check availability of tools (shadcn, motion, gsap).
2. **Phase 1**: Design the motion script (e.g., "Hero text explodes, image spins").
3. **Phase 2**: Implement with separation of concerns (Logic vs UI).

## Tech Stack
- **Framework**: Next.js App Router
- **Styling**: Tailwind CSS + CSS Variables
- **Motion**: 
  - *Marketing*: GSAP ScrollTrigger
  - *Product*: Framer Motion
- **Components**: Shadcn UI (Radix Primitives)
